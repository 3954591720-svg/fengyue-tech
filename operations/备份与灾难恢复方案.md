# 备份与灾难恢复方案

> 版本：v1.0
> 生效日期：2026-07-24
> 适用范围：宝鸡丰悦云创科技有限公司所有生产系统
> 核心目标：RTO ≤ 4 小时 / RPO ≤ 24 小时

---

## 一、备份对象

### 1.1 代码仓库
- **位置**：GitHub（公网 + 私有）
- **策略**：
  - 公网仓库：`3954591720-svg/fengyue-tech`（官网）
  - 私有仓库：`3954591720-svg/fengyue-yunchuang-platform`（商业代码）
- **备份方式**：GitHub 自带冗余 + 本地 weekly 镜像克隆
- **保留期限**：永久
- **RPO**：0（实时）
- **RTO**：≤ 30 分钟（克隆 + 部署）

### 1.2 客户业务数据（数据库）
- **类型**：MySQL / PostgreSQL / MongoDB / Redis
- **备份方式**：
  - 每日凌晨 3:00 全量备份（mysqldump / pg_dump）
  - 每 6 小时增量备份（binlog / WAL）
  - 实时备份至 OSS（阿里云）
- **保留期限**：30 天滚动
- **异地副本**：阿里云 OSS（华东1）+ 腾讯云 COS（华南）
- **RPO**：≤ 6 小时（增量丢失容忍）
- **RTO**：≤ 2 小时（数据库恢复 + 应用启动）

### 1.3 文件存储
- **类型**：用户上传的图片、视频、文档、备份文件
- **备份方式**：
  - 每周日全量同步至 OSS
  - 实时增量同步（ossutil / coscli）
- **保留期限**：永久（业务文件）/ 90 天（临时文件）
- **RPO**：≤ 7 天
- **RTO**：≤ 1 小时

### 1.4 配置文件与密钥
- **位置**：`/etc/` `/opt/` `.env`
- **备份方式**：
  - 加密后推送至 GitHub 私有仓库（独立分支）
  - 加密密钥由 1Password 管理
- **保留期限**：永久
- **RPO**：0
- **RTO**：≤ 30 分钟

### 1.5 服务器镜像
- **频率**：每月 1 日 / 重大变更后
- **方式**：阿里云快照 / 自定义镜像
- **保留**：最近 3 个
- **RTO**：≤ 30 分钟（基于镜像启动新实例）

## 二、备份策略详情

### 2.1 自动备份脚本（cron）

```bash
# 数据库每日全量
0 3 * * * /opt/scripts/backup_db.sh

# 数据库每 6 小时增量
0 */6 * * * /opt/scripts/backup_db_incremental.sh

# 文件每周日全量
0 4 * * 0 /opt/scripts/backup_files.sh

# 镜像备份
0 5 1 * * /opt/scripts/backup_image.sh

# 配置每周一同步
0 6 * * 1 /opt/scripts/backup_config.sh

# 备份验证（每月最后一天）
0 7 L * * /opt/scripts/verify_backup.sh
```

### 2.2 备份验证

每月最后一天执行：
1. 随机抽取最近 7 天的一个备份
2. 在隔离环境恢复
3. 校验数据完整性（行数 / 校验和）
4. 输出验证报告

### 2.3 备份监控

- 备份成功率 < 95% 触发告警（邮件 + 短信）
- 备份文件大小异常（偏离均值 ± 50%）触发告警
- 备份超过 24 小时未执行触发告警

## 三、灾难恢复预案

### 3.1 灾难分级

| 级别 | 场景 | 影响 | 恢复目标 |
|------|------|------|---------|
| D0 | 单文件误删 / 单库表损坏 | 局部 | RTO ≤ 30 分钟 |
| D1 | 单服务器故障 | 局部 | RTO ≤ 1 小时 |
| D2 | 数据库完全损坏 | 严重 | RTO ≤ 2 小时 |
| D3 | 单机房不可用 | 严重 | RTO ≤ 4 小时 |
| D4 | 多机房不可用 / 数据泄露 | 灾难 | RTO ≤ 24 小时 |

### 3.2 恢复步骤（D2 数据库损坏示例）

```
T+0:00  事件发生
T+0:05  启动应急响应，通知法人
T+0:15  评估影响范围，确定恢复点（RPO）
T+0:30  启动备用数据库实例
T+1:00  从 OSS 拉取最近全量备份
T+1:30  恢复全量备份
T+1:45  重放增量日志（binlog）
T+2:00  验证数据完整性
T+2:15  应用切换至新数据库
T+2:30  监控验证
T+4:00  复盘总结
```

## 四、备份存储安全

### 4.1 加密
- 备份文件传输：TLS 1.3
- 备份文件存储：AES-256-GCM
- 加密密钥：1Password / 阿里云 KMS
- 密钥轮换：每年 1 次

### 4.2 访问控制
- 备份存储使用独立 RAM 账号
- 仅法人 + 技术负责人有访问权限
- 操作日志保留 5 年

### 4.3 异地容灾
- 主备机房物理隔离（≥ 100 公里）
- 阿里云华东1 ↔ 阿里云华北2
- 腾讯云华南 ↔ 腾讯云西南

## 五、恢复演练

### 5.1 演练频率
- **每季度一次**：单文件/单库恢复演练
- **每半年一次**：单服务器故障恢复演练
- **每年一次**：跨机房灾难恢复演练

### 5.2 演练流程
1. 提前 7 天通知法人审批
2. 在隔离测试环境执行
3. 全程记录（截图、日志、时间戳）
4. 输出《演练报告》
5. 整改发现的问题
6. 归档报告（保存 5 年）
