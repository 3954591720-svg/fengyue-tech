# 宝鸡丰悦云创科技公司 · 客户沟通 SOP 与话术模板

> 适用场景：创始人直客、软件定制开发 / 技术服务 / 小程序 / APP / 网站  
> 核心理念：专业 + 真诚 + 有温度，不套路，不忽悠

---

## 一、初始沟通流程（从加微信到成交）

### 流程总览

```
① 客户主动添加微信 / 被推荐
      ↓
② 24小时内通过，发送开场白
      ↓
③ 寒暄破冰，了解基本信息
      ↓
④ 深度挖掘需求（问对问题）
      ↓
⑤ 出方案或报价（阶梯式）
      ↓
⑥ 处理异议 / 讨价还价
      ↓
⑦ 成交确认 & 预付款
      ↓
⑧ 正式立项，进入项目流程
```

### 每步详细说明

#### ① 客户添加好友

- **通过时间：** 尽量在 2 小时内通过；超过 12 小时必须补一句"抱歉久等了"
- **添加来源标注：** 微信备注好"客户-来源渠道-日期"，例：`客户-抖音咨询-0723`

#### ② 开场白（客户添加后 5 分钟内发送）

```
您好！感谢添加，我是丰悦云创的阿丰，专注于软件开发和技术服务。

请问怎么称呼您？方便说一下您想做的项目大概是什么方向吗？
```

#### ③ 寒暄破冰

```
核心原则：先聊人，再聊事。不要一上来就问技术细节。

示例：
- "听朋友说您在做XXX，最近忙吗？"（有推荐人时）
- "是朋友推荐过来的，还是看到我们的案例找到的？"（无推荐人时）
- "您的行业我挺感兴趣的，平时主要做什么业务？"
```

#### ④ 需求挖掘（见第二章节话术模板）

#### ⑤ 报价

```
报价前必做：
□ 确认清楚需求范围
□ 确认交付时间和紧急程度
□ 确认是否需要长期维护
□ 确认客户预算预期（先问"您这边大概预算范围是多少？"）
```

#### ⑥ 异议处理（见第二章节）

#### ⑦ 成交确认

```
确认成交后发送：
您的项目需求我们已经整理好了，下面是确认事项：

📋 项目内容：[简述需求]
💰 报价金额：[X] 元
📅 交付周期：[X] 个工作日
💳 付款方式：预付 50%，交付验收后结清余款
🔧 维护期：交付后 [X] 个月内免费维护

确认无误后我们即可开始，我会第一时间拉项目群。
稍后我会发一份正式报价单给您，方便您内部走流程。
```

#### ⑧ 预付款到账后，正式立项

## 二、需求沟通话术模板

### 场景 1：客户说"帮我做个网站 / APP / 小程序"

**错误示范：**
> "好的，您想要什么样的？"

**正确思路：引导客户说出真实需求，而不是猜测。**

**推荐话术：**

```
好的，完全理解！这类需求我可以做，交付过不少。

为了给您一个准确的方案，我想先了解几个关键问题：

1. 【做什么用】
   "这个网站/小程序，主要解决您什么问题？
   是展示型为主，还是需要用户注册、下单、支付这类功能？"

2. 【给谁用】
   "目标用户是谁？是您的客户直接用，还是内部员工用？"

3. 【内容规模】
   "大概有多少内容需要展示？比如产品页面、新闻文章，
   预计有多少个？有没有参考的网站/小程序样式？"

4. 【上线时间】
   "您希望什么时候上线？有没有比较紧迫的时间节点？"

5. 【预算范围】
   "方便透露一下预算范围吗？这样我可以给您匹配最合适的方案。"
```

**黄金法则：问"做什么用"比问"做什么"更重要。**  
**多问功能目的，少问视觉风格；先问业务逻辑，后问技术实现。**

---

### 场景 2：客户问"多少钱"

**核心原则：**

- 不报虚价（报高价再大折扣，显得不真诚）
- 不报底价（一上来就报最低价，没余地也没利润）
- 报合理价 + 说明构成 + 留议价空间

**推荐话术（客户直接问价）：**

```
完全理解，价格是核心关注点。
在报价之前，我想先跟您确认一下需求范围，这样给您报出来的数字是准确的，不是那种先报低再中途加价的方式。

您看方便说说：
1. 您的业务场景是什么？
2. 需要哪些核心功能？
3. 有没有预算区间参考？

我这边整理好方案后，会给您一个清晰的分项报价，每一项是什么功能、多少钱，您拿到手一目了然。
```

**当客户预算太低时（温和拒绝）：**

```
感谢您的坦诚，我理解您的预算考量。

说实话，按照您刚才说的功能需求，这个价格目前做不下来。
不是说不行，是做出来的质量可能达不到您的预期，这样对双方都不好。

我有几个建议：
① 我们可以精简一些非核心功能，先把最关键的功能做出来，后续再迭代；
② 如果时间不急，我可以帮您规划一个分阶段方案，先解决核心需求；
③ 您也可以再看看其他选择，找到最符合您预期的方案。

不管最终合作与否，我这边都乐意给您一些行业建议，供您参考。
```

---

### 场景 3：客户说"我先想想"

**核心原则：不要逼单，但要留钩子。**

**推荐话术：**

```
完全理解，这是大事，多考虑一下是应该的。

我这边把刚才聊的需求整理了一份简要文档，您方便的时候可以看看，心里有个底。

有一点想提醒您：这个项目早点启动，上线时间就能更有保障，报价我现在给您的也是当前的价格，后续材料成本和工时可能会浮动。

您考虑期间，如果有任何疑问随时找我，不用客气。
不打扰您，祝您一切顺利！
```

**跟进策略（不要连续轰炸）：**

| 时间节点 | 动作 | 话术参考 |
|---|---|---|
| 当天 | 发需求确认文档 | "这是我整理的需求摘要，方便您对照参考。" |
| 3天后 | 轻量级问候 | "项目进展怎么样了？有没有我能帮上忙的地方？" |
| 7天后 | 价值推送 | "上周有个客户和您需求类似，我们刚交付了，给您分享一下案例。" |
| 14天后 | 最后一次 | "想再问一下，项目这边是什么情况？需要我这边再补充什么吗？" |

---

### 场景 4：客户说"别人比你便宜"

**推荐话术：**

```
价格差异这个现象很正常，我也不想跟您打价格战。

我只能说说我们这边的情况，供您参考：

✅ 我们是创始人直接对接，不是交给外包或实习生
✅ 需求沟通→开发→测试→上线，全程可追溯，您随时了解进度
✅ 交付后有维护期，出了问题找得到人
✅ 代码规范，后续迭代成本低

说实话，我见过不少客户图便宜做了，后来发现功能烂、维护难、二次开发费用比第一次还贵。

您也可以让对方给您出一个详细的功能清单和验收标准，对比一下我们两家的内容再做决定，您觉得呢？
```

## 四、项目交付流程

### 交付总流程

```
阶段一：启动确认（Day 1-3）
  ├── 签署合同 & 收取预付款
  ├── 建立项目沟通群（微信 + 备选飞书/钉钉）
  ├── 召开项目启动会（线上，30分钟）
  └── 输出：《项目启动确认书》

阶段二：需求分析 & 原型设计（Week 1-2）
  ├── 详细需求调研（功能清单、页面结构、交互逻辑）
  ├── 输出：《功能需求文档》《页面结构图》
  ├── 原型设计（低保真原型）
  ├── 客户评审 & 确认签字
  └── 输出：《确认版原型》← 重要里程碑！

阶段三：UI设计（Week 2-3）
  ├── 高保真设计稿（视觉稿）
  ├── 客户评审（2轮免费修改，超出按工时计）
  ├── 设计定稿签字
  └── 输出：《设计定稿稿》

阶段四：功能开发（Week 3-8，根据项目调整）
  ├── 前端开发
  ├── 后端开发
  ├── 第三方接口对接（支付、短信等）
  ├── 每日/每周进度同步（截图或录屏）
  └── 输出：可运行的测试版本

阶段五：测试 & 修复（Week 8-9）
  ├── 内部测试（功能测试、性能测试、兼容性测试）
  ├── 输出：《测试报告》
  ├── Bug 修复
  └── 提供测试账号给客户体验

阶段六：客户验收（Week 9-10）
  ├── 客户提供验收清单，逐项确认
  ├── 小问题现场修复
  ├── 大问题记录排期
  ├── 客户签署《验收确认书》
  └── 收取验收款

阶段七：上线部署（Day 上线当天）
  ├── 服务器/域名配置
  ├── 正式环境部署
  ├── 数据迁移（如有）
  ├── 监控报警配置
  └── 正式上线公告

阶段八：交付 & 售后（上线后）
  ├── 交付所有源代码、设计稿、文档
  ├── 操作培训（远程，30分钟）
  ├── 签署《项目交付确认书》
  └── 进入维护期
```

### 关键节点检查清单

**【里程碑1】需求确认签字**
- [ ] 功能清单客户已确认
- [ ] 原型客户已确认（签字或截图确认）
- [ ] 任何变更已记录并更新报价

**【里程碑2】设计定稿签字**
- [ ] UI稿客户已确认（不超过2轮大改）
- [ ] 超出修改范围已沟通并确认额外费用

**【里程碑3】测试版本验收**
- [ ] 客户已在测试环境完整体验
- [ ] 所有Bug已记录并分类（P0紧急/P1重要/P2常规）
- [ ] P0/P1级Bug已修复完成

**【里程碑4】正式上线**
- [ ] 预验收款已到账
- [ ] 验收确认书已签署
- [ ] 所有交付物已打包移交

## 五、售后跟进话术

### 5.1 交付后 1 周

**目标：确认客户已正常使用，收集初始反馈**

```
Hi [客户名]，项目上周已经正式上线了，辛苦了！

想问一下：
1. 日常使用流程上有没有觉得不顺手的地方？
2. 有没有发现什么问题或者 Bug？
3. 需要我给您做一次操作培训吗？半小时就够了。
```
