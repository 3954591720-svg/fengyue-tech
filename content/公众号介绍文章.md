### 在宝鸡，一个人做软件公司是什么体验

---

凌晨一点，键盘声在安静的办公室里格外清晰。

屏幕上的代码跑了三遍，终于把那个折磨了两天的接口 bug 修掉了。我靠在椅背上，喝了口已经凉透的茶，窗外是宝鸡老城区安静的夜色。

这大概是很多人想象中"自由职业者"不会有的画面——但这就是我的日常。

我是丰悦云创的创始人，也可能是你见过的最不像老板的软件工程师。

---

#### 为什么在宝鸡，一个人做软件公司？

2022年，我从西安回到了宝鸡。

在外漂泊了十几年，写过金融系统，做过电商平台，也在创业公司带过团队。技术栈从后端到前端，从服务器到数据库，基本都摸了个遍。35岁那年，我决定不再租房搬家，回宝鸡，用自己的方式做点事情。

很多人问我：为什么不去北上广？为什么要回宝鸡？

我想了想，答案是：因为我受够了"大公司病"。

一个需求，从产品到开发到测试到运维，绕了一大圈，最后做出来的东西和最初的想法已经差了十万八千里。更让人疲惫的是，当你在一个流程里只是一个环节，你永远无法真正对一个产品、对一个客户负责。

所以我注册了**宝鸡丰悦云创科技公司**。

不是什么宏大的梦想，就是想做一个**靠谱的技术合伙人**——不画饼，不吹牛，亲自写代码，亲自跟客户对接，把事情做好。

---

#### 一个人，能做什么？

说实话，刚开业那阵子我也心虚：一个人做公司，客户会不会觉得不靠谱？

后来我发现，恰恰是"一个人"，反而成了最大的优势。

**没有中间商，没有扯皮，没有汇报链。**

客户的需求，我直接听、马上消化、自己动手做。做完了直接交付，测试没问题就上线。全程就我一个人盯着，没有哪个环节会掉链子。

目前我能承接的业务主要包括：

- **软件开发定制**：桌面端/Web端/移动端均可，从需求梳理到上线维护全流程
- **技术咨询与方案**：系统架构设计、技术选型、代码审查，现有系统的问题诊断
- **运维与部署**：Linux 服务器维护、Docker 容器化、自动化部署流程
- **API 接口开发**：RESTful / GraphQL 接口设计与实现，第三方系统对接

技术栈方面，后端我主用 Java / Spring Boot / Python / Go，前端 Vue / React 都能搞，数据库 MySQL / PostgreSQL / Redis 都不陌生。App 开发和小程序也有实战经验。

一句话：**全栈开发，没有短板**。

---

#### 为什么坚持一个人？

有人问我：你就不想做大吗？

说实话，想过。但冷静下来想想，扩张意味着我要花大量时间去招人、管理、开会——而这些时间，原本可以用来给客户写代码。

我选择一个人，不是因为做不大，而是因为**我更想把时间花在真正有价值的事情上**：研究技术、解决客户的问题、把产品打磨好。

一个人的公司，听起来很孤独。

但其实，我的客户从来不是"甲方爸爸"，而是**一起解决问题的合伙人**。我们坐下来聊需求，我帮你拆解成技术方案，然后我把东西做出来，整个过程干净、透明、高效。

这种关系，比在大公司里被流程推着走舒服多了。
